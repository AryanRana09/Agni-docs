### Overview

Use this endpoint to **retrieve a single agent** by its ID for a specific organization.

The base URL for all requests from the playground is `https://api.ravan.ai/api/v1`.

### Authorization

- **Authorization**: `Bearer YOUR_API_KEY` (header, required)  
  Example: `Authorization: Bearer sk_live_xxx`

### HTTP request

- **Method**: `GET`  
- **Path**: `/agents/{id}`

### Path parameters

| Name | Type  | Required | Description                | Example |
| ---- | ----- | -------- | -------------------------- | ------- |
| `id` | int64 | Yes      | Unique identifier of agent | `123`   |

### Query parameters

| Name              | Type   | Required | Description                                      | Example          |
| ----------------- | ------ | -------- | ------------------------------------------------ | ---------------- |
| `organizationId`  | string | Yes      | Organization identifier that owns the agent.     | `"org_123"`      |

### Successful response (200)

On success, the API returns the requested agent.

#### Top-level fields

| Field     | Type   | Description                                   |
| --------- | ------ | --------------------------------------------- |
| `success` | boolean | Indicates whether the request was successful. |
| `message` | string  | Human-readable message about the operation.   |
| `data`    | Agent   | Agent details.                                |

#### `Agent` object

The Agent object corresponds to the `Agent` message in the service definition.

Key fields:

- **id** (int64): Unique identifier of the agent.  
- **organization_id** (string): Identifier of the owning organization.  
- **widget_settings_id** (int64, optional): Associated widget settings configuration.  
- **name** (string, optional): Internal name of the agent.  
- **agent_name** (string, optional): Display name of the agent.  
- **status** (enum): Current status of the agent: `AGENT_STATUS_ACTIVE` or `AGENT_STATUS_INACTIVE`.  
- **model**, **s2s_model**, **voice_id** (string, optional): Model and voice configuration.  
- Additional configuration fields controlling temperature, reminders, ambient sound, call behavior, IVR options, tool selection, integrations, knowledge base, prompts, and timestamps.
