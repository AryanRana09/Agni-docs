---

### Overview

Use this endpoint to **create a telephony configuration** that defines how calls are routed through an external provider.

The base URL for all requests from the playground is `https://api.ravan.ai/api/v1`.

### Authorization

- **Authorization**: `Bearer YOUR_API_KEY` (header, required)  
  Example: `Authorization: Bearer sk_live_xxx`

### HTTP request

- **Method**: `POST`  
- **Path**: `/create/telephony`

### Request body

Send a JSON object with the following fields.

| Field           | Type   | Required | Description                                                       | Example             |
| --------------- | ------ | -------- | ----------------------------------------------------------------- | ------------------- |
| `type`          | string | Yes      | Telephony type identifier (for example, `"sip"` or `"pstn"`).    | `"sip"`             |
| `owner_type`    | string | Yes      | Type of owner associated with this telephony configuration.       | `"organization"`    |
| `friendly_name` | string | Yes      | Human-readable name for the telephony configuration.             | `"Primary SIP"`     |
| `client_id`     | string | Yes      | Client ID used to authenticate with the telephony provider.      | `"client_123"`      |
| `client_secret` | string | Yes      | Client secret used to authenticate with the telephony provider.  | `"secret_123"`      |

<RequestExample>

```bash 201 - Create telephony type
curl --request POST \
  --url https://api.ravan.ai/api/v1/create/telephony \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "type": "sip",
    "owner_type": "organization",
    "friendly_name": "Primary SIP",
    "client_id": "client_123",
    "client_secret": "secret_123"
  }'
```

</RequestExample>

### Successful response (201)

On success, the API returns the created telephony configuration.

#### Top-level fields

| Field     | Type          | Description                                   |
| --------- | ------------- | --------------------------------------------- |
| `success` | boolean       | Indicates whether the request was successful. |
| `message` | string        | Human-readable message about the operation.   |
| `data`    | TelephonyType | Telephony configuration details.              |

#### `TelephonyType` object

| Field            | Type   | Description                                                   | Example               |
| ---------------- | ------ | ------------------------------------------------------------- | --------------------- |
| `id`             | string | Unique identifier for the telephony configuration.           | `"tel_123"`           |
| `type`           | string | Telephony type identifier.                                   | `"sip"`               |
| `owner_type`     | string | Type of owner associated with this telephony configuration.  | `"organization"`      |
| `friendly_name`  | string | Human-readable name for the telephony configuration.         | `"Primary SIP"`       |
| `status`         | string | Current status of the telephony configuration.               | `"active"`            |
| `created_at`     | string | ISO 8601 timestamp when the configuration was created.       | `"2024-01-01T00:00:00Z"` |
| `termination_uri`| string | Default termination URI, if available.                       | `"sip:support@example.com"` |
| `sip_username`   | string | SIP username used for this configuration, if applicable.     | `"support-user"`      |
| `sip_password`   | string | SIP password used for this configuration, if applicable.     | `"strong-password"`   |
